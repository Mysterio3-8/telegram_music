# SPEC — Telegram Music Bot v1.0

Полное ТЗ. Источник истины по функциональности. Не редактировать без решения владельца проекта.

## 1. Цель

Telegram-бот для поиска, хранения, организации и загрузки музыкальных треков.
Полноценная музыкальная платформа внутри Telegram с развитием в Mini App.

Требования: простой интерфейс, высокая скорость, масштабируемая архитектура,
автоматическое наполнение базы, отсутствие ручной модерации.

## 2. Функции

Поиск музыки, поиск минусов, личная библиотека, плейлисты, загрузка своих треков,
скачивание, шеринг, Premium-подписка, Mini App (позже).

## 3. Главное меню (`/start` — личный кабинет)

```
👋 Добро пожаловать!

Имя:
Telegram ID:

Подписка: Бесплатная

Треков в библиотеке: 0
Плейлистов: 0

Выберите действие:
```

Кнопки: 🔍 Поиск треков | 🎵 Библиотека | 📂 Плейлисты | 🎼 Поиск минусов |
⬆️ Загрузить трек | 💎 Купить Premium | 🌐 Mini App

## 4–6. Библиотека

- Личная, после регистрации пустая. Попадают: добавленные и загруженные треки.
- Экран: `🎵 Библиотека / Всего треков: N`, далее список — ровно 5 треков на страницу.
- Навигация: `⬅️ Страница X / Y ➡️`, `🔍 Найти в библиотеке`, `🎲 Случайный трек`, `◀️ Назад`.
- Поиск в библиотеке: запрос «Введите название», результаты только из библиотеки пользователя.
- Случайный трек: случайный из библиотеки пользователя.

## 7–8. Плейлисты

- Любое количество (лимит для бесплатной версии — настраиваемый параметр).
- Экран: `📂 Плейлисты / Всего: N`, по 5 на страницу, `➕ Создать плейлист`, `◀️ Назад`.
- Просмотр плейлиста: название, всего треков, список по 5, пагинация, назад.

## 9–10. Поиск треков и минусов

- «Введите название трека» → результаты по 5 (оригинал, cover, remix, live, acoustic...).
- Поиск минусов полностью повторяет поиск треков.

## 11. Карточка трека

Название, исполнитель, длительность.
Кнопки: ➕ Добавить в библиотеку | 📂 Добавить в плейлист | 📤 Поделиться | 🗑 Удалить.
Из библиотеки — «Удалить из библиотеки», из плейлиста — «Удалить из плейлиста».
Трек никогда не удаляется из общей базы.

## 12–13. Загрузка трека и автомодерация

Мастер: аудиофайл → название → исполнитель → подтверждение.
Автоматически: проверка дубликатов, анализ метаданных, аудиоотпечаток, тип записи.
Дубликат → «Такой трек уже есть. Добавить его в вашу библиотеку?».
Новый → в общую базу + в библиотеку пользователя.
Ручной модерации нет. Совпадает название, но отличается отпечаток → новая версия (remix/live/instrumental).

## 14. Premium

Цена 17 ₽. Оплата: Telegram Stars, карта/СБП. Активация автоматическая.
Даёт: без рекламы, безлимит плейлистов, увеличенный лимит загрузок, ранний доступ, поддержка проекта.

## 15. Mini App

Пока: «Mini App находится в разработке.» Позже — приложение на той же БД и API.

## 16, 23. Общая база и импорт

Единая база: треки, минусы, исполнители, загрузки, плейлисты, библиотеки.
Модуль импорта: подключается к разрешённым источникам, пополняет каталог,
обновляет метаданные, грузит файлы в хранилище, создаёт отпечатки, убирает дубликаты.
Работает в фоне, не влияет на пользователей.

## 17–18. Архитектура и стек

```
Telegram Bot → Telegram Bot API → Backend API
                                      ├── PostgreSQL (данные)
                                      ├── Redis (кэш, FSM, rate limit, очереди)
                                      └── S3-совместимое хранилище (MinIO / R2 / B2) — mp3, flac, wav, минусы
```

Сервисы: поиск, импорт каталога, проверка дубликатов, оплата Premium, API для Mini App.
Стек: Python 3.13+, FastAPI, aiogram 3.x, SQLAlchemy, Alembic, Celery/Dramatiq.

## 19. Структура БД

- **users**: id, telegram_id, username, first_name, language, premium, premium_until, created_at, last_login
- **tracks**: id, title, artist, album, duration, bitrate, file_size, format, storage_path, fingerprint, created_at
- **instrumentals**: id, title, artist, duration, storage_path, fingerprint
- **playlists**: id, user_id, title, created_at
- **playlist_tracks**: playlist_id, track_id, position
- **user_library**: user_id, track_id, added_at
- **premium**: user_id, status, type, start_date, end_date, payment_id
- **uploads**: id, user_id, track_id, upload_date

## 20. Навигация

Только Inline Keyboard. Каждая страница: ⬅️ ➡️ ◀️ Назад.
Первая страница — ⬅️ отключается, последняя — ➡️ отключается.

## 21–22. Ограничения и проверка файлов

Максимум 50 МБ (настраиваемо). Форматы: MP3, FLAC, WAV, M4A, OGG.
Проверка: расширение, размер, длительность, повреждение, воспроизводимость.
Извлекаются: название, исполнитель, длительность, битрейт, размер, формат. Создаётся отпечаток.

## 24. Реклама

Для бесплатных: после каждого N-го действия (по умолчанию 10, настраиваемо).
Premium отключает полностью.

## 25. Логи

Вход, ошибки, загрузки файлов, оплата, создание плейлистов, удаление треков,
импорт каталога и его ошибки, ошибки оплаты.

## 26. Админ-панель (веб)

Пользователи, счётчики треков/загрузок/Premium, логи, ошибки, ручной запуск импорта,
статистика, управление рекламой, ценой Premium, настройками бота.

## 27. API (REST, JWT)

POST /login, GET /tracks, GET /track/{id}, GET /instrumentals, POST /upload,
POST /playlist, POST /payment, GET /library, GET /search, GET /random.
Готовность к подключению Mini App.

## 28. Производительность

100 000 пользователей, 10 000 одновременно, каталог 100M+ треков, горизонтальное масштабирование.

## 29. Безопасность

HTTPS, валидация входных данных, rate limit, защита от SQL-инъекций и XSS,
регулярные бэкапы, разграничение прав пользователь/админ.
