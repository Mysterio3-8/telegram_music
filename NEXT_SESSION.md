## P1. Плеер пишет «трек недоступен» — починить навсегда

**Требование владельца:** треки должны всегда быть доступны и играть. Хранить
все файлы на диске НЕ обязательно (не засорять диск) — но стрим должен работать
стабильно.

**Диагностика уже сделана, две подтверждённые причины:**

1. **Протухшие подписанные ссылки.** `audio_url` подписывается на 6 часов
   (`AUDIO_URL_TTL_SECONDS = 6*3600` в [app/api/security.py](app/api/security.py)).
   Фронт кэширует треки с их `audio_url` в localStorage: «Недавние»
   (`prefs.js: pushRecentTrack`) и офлайн-индекс. Через 6+ часов кэшированная
   ссылка → 403 «Ссылка недействительна или истекла» → плеер говорит
   «Не удалось загрузить трек — пропускаю» ([state.js](miniapp/src/state.js):110).
   **Фикс:** фронту при 403/ошибке аудио перезапрашивать свежий трек по id
   (`GET /tracks/{id}` или обновлять audio_url перед проигрыванием кэшированных),
   а не скипать. Плюс не хранить audio_url в prefs вовсе — только id/title/artist,
   URL резолвить при старте воспроизведения.

2. **Лимит Bot API 20 МБ.** Треки, заминченные через бота (YouTube/SoundCloud/ТГ-канал),
   не имеют `storage_path` — эндпоинт [audio.py](app/api/routers/audio.py) качает
   байты у Telegram через `bot.download(tg_file_id)`. Bot API не отдаёт файлы
   больше 20 МБ → такие треки в Mini App вечно «недоступны» (в боте при этом
   играют нормально — там пересылка file_id).
   **Фикс-варианты:** (а) воркер обогащения докачивает такие треки в хранилище
   (S3/local) в фоне — тогда стрим всегда из архива; (б) кэш «горячих» треков
   на диске с LRU-вытеснением (владелец: «не засорять диск» — LRU-кэш на N ГБ
   это ровно оно); (в) для >20 МБ качать заново из первоисточника. Рекомендация:
   (а)+(б): фоновое докачивание всего, LRU-чистка старого.

Проверить также: `bot.download` на каждый плей без кэша = медленный старт — добавить
кэширование байтов (диск, LRU) в `_load_audio_bytes`.

## P2. Прокси для массового скачивания SoundCloud 24/7

**Требование:** владелец будет лить МНОГО треков (24/7). Нужны прокси, чтобы IP
сервера не банили. Задержка между скачиваниями — **рандом 5–60 секунд**
(сейчас 2–5 сек в [soundcloud_import.py](app/services/soundcloud_import.py):
`_MIN_DELAY/_MAX_DELAY` — поменять и вынести в конфиг).

**Сделать:**
- Конфиг `PROXY_LIST` (список http/socks5-прокси через запятую) или файл.
- yt-dlp опция `proxy` в `_base_opts()` — ротация по кругу/рандому на каждый
  запрос; при ошибке через прокси — пробовать следующий.
- Задержка `SOUNDCLOUD_MIN_DELAY=5` / `SOUNDCLOUD_MAX_DELAY=60` (env).
- ⛔ Сами прокси-адреса даст владелец (или купить — спросить какие: желательно
  резидентные/mobile, датацентровые SoundCloud режет быстрее).
- Источники для массовой закачки владелец скинет или добавит сам через админку.

## P4. Артисты как сущности + чистка Unknown

**Требование:** владелец пришлёт **текстом** список артистов ⛔. К каждому
привязать треки и альбомы. После привязки — **удалить все треки, у которых
артист остался Unknown/«неизвестный»**.

План реализации:
- Таблица `artists` (id, name, normalized_name) + `tracks.artist_id` (FK, nullable
  на переходный период) — миграция. `/artists` начинает читать таблицу.
- CLI/админка: маппинг «текстовый список владельца» → artists; треки биндить
  по `lower(trim(tracks.artist))`, альбомы — по `tracks.album` тех же треков.
- Чистка: треки с artist в ('Unknown', 'неизвестный', '') и без artist_id —
  удалить полностью (механика удаления уже есть в
  [catalog_cleanup.py](app/services/catalog_cleanup.py) — переиспользовать связи-чистку).
  ⚠️ Это второе исключение из инварианта «трек не удаляется» — подтвердить у
  владельца список перед удалением (показать счётчик и примеры).

1. **Вход в бота 1 ₽** («проверка на бота/спам», без привязки карты, один раз):

     ставим `users.entry_paid=true` и пускаем дальше. Миграция + гейт в
     SubscriptionMiddleware-стиле (или в том же).
   - Существующих юзеров НЕ трогать (grandfather: `entry_paid=true` бэкфиллом).
   - ⚠️ Обсудить в начале сессии: комиссия ЮKassa на 1 ₽ съедает заметную долю;
     альтернатива — Telegram Stars (1 ⭐), где UX в один тап без выхода из TG.
